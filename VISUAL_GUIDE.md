## ğŸ“ Typography

### Font Families

**Playfair Display** (Serif)
- Used for: Headings, titles, hero text
- Weights: 400, 600, 700, 800
- Style: Elegant, sophisticated, classic

**Poppins** (Sans-serif)
- Used for: Body text, buttons, navigation
- Weights: 300, 400, 500, 600
- Style: Modern, clean, readable

### Typography Scale
```
Hero Title:        64px / 4rem    (Playfair Display, Bold)
Page Title:        48px / 3rem    (Playfair Display, Bold)
Section Heading:   36px / 2.25rem (Playfair Display, SemiBold)
Card Title:        24px / 1.5rem  (Playfair Display, SemiBold)
Body Large:        18px / 1.125rem (Poppins, Regular)
Body:              16px / 1rem    (Poppins, Regular)
Body Small:        14px / 0.875rem (Poppins, Light)
Caption:           12px / 0.75rem (Poppins, Light)
```

## ğŸ¨ Design Principles

### 1. Glassmorphism First
Every card, section, and interactive element uses the frosted glass effect for visual consistency and premium feel.

### 2. Hierarchy Through Size
```
Most Important:    Large text, bold weight, accent color
Important:         Medium text, semibold weight
Supporting:        Regular text, lighter weight
Least Important:   Small text, light weight, muted color
```

### 3. Consistent Spacing
```
Section Padding:   80px (desktop) / 40px (mobile)
Card Padding:      32px (desktop) / 20px (mobile)
Element Margin:    24px between elements
Grid Gap:          32px (desktop) / 20px (mobile)
```

### 4. Color Usage
```
Primary (Indigo):     Backgrounds, headers
Accent (Purple):      CTAs, highlights, hover states
White:                Primary text, icons
Light Gray:           Secondary text
Transparent Glass:    All cards and overlays
```

### 5. Interactive Feedback
Every interactive element provides visual feedback:
- Buttons: Color change + slight scale
- Cards: Lift effect + border glow
- Links: Color transition
- Inputs: Border glow on focus

## ğŸ¯ Performance Optimization

### Image Optimization
```
Strategy:
1. Use WebP format with JPG fallback
2. Lazy load images below fold
3. Responsive images (srcset)
4. Compress to 85% quality
5. Serve appropriate sizes

Example:
<img 
  src="image.webp"
  srcset="image-400.webp 400w,
          image-800.webp 800w,
          image-1200.webp 1200w"
  sizes="(max-width: 768px) 100vw,
         (max-width: 1024px) 50vw,
         33vw"
  loading="lazy"
  alt="Description"
/>
```

### CSS Optimization
```
1. Minimize CSS file size
2. Use CSS variables for consistency
3. Avoid expensive properties (box-shadow on scroll)
4. Use transform for animations (GPU accelerated)
5. Combine similar styles
```

### JavaScript Optimization
```
1. Code splitting by route
2. Lazy load components
3. Debounce scroll/resize events
4. Use Vue 3 Composition API
5. Minimize bundle size
```

### Font Loading
```
Strategy:
1. Preload critical fonts
2. Use font-display: swap
3. Subset fonts (Latin only)
4. Load weights actually used

<link rel="preload" 
      href="fonts/playfair.woff2" 
      as="font" 
      type="font/woff2" 
      crossorigin>
```
